Rewards API - Polkadot

Authentication & Making Calls​

Making API calls with DataHub requires authentication, which can be performed by passing your API key via the request header Authorization or as a parameter in the query URL.

Here's some boilerplate to get you started.

  • Fetch with Header
  • Fetch with Parameter
  • CURL with Header
  • CURL with Parameter
fetch("https://polkadot-rewards.datahub.figment.io/rewards", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Authorization" : "<api_key>",
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
fetch("https://polkadot-rewards.datahub.figment.io/rewards", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
curl -X POST 'https://polkadot-rewards.datahub.figment.io/rewards' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // JSON Payload
}'
curl -X POST 'https://polkadot-rewards.datahub.figment.io/rewards' \
--header 'Content-Type: application/json' \
--data '{
  // JSON Payload
}'

Available Methods

Rewards (by epoch)#

Returns net delegator rewards (aka post on-chain validator commission) grouped by epoch

View specification

Request

  • network => polkadot
  • chain_id => mainnet
  • accounts takes in an array of Stash Account public addresses
  • grouping => epoch will return rewards grouped by eras.
  • start_time and end_time are ISO 8601 timestamps in UTC. If the start of an epoch falls into the requested date range it will be included (even if it doesn't end inside the date range)

Example:

curl --location --request GET 'https://polkadot-rewards.datahub.figment.io/rewards' \
-H 'apikey: $APIKEY' \
-H "Content-Type: application/json" \
--data-raw '{
    "network": "polkadot",
    "chain_id": "mainnet",
    "accounts": [
        "12m7XtVWZQuxTirqtzm2dm5J4r16vSQruPmahThMnskG8Bcp"
    ],
    "start_time": "2022-07-10",
    "end_time": "2022-07-17",
    "grouping": "epoch"
}'

Response

  • network and chain_id are the same as the request
  • accounts returns an array of Stash Account public addresses
  • validator returns the validator that tokens were staked to
  • time and epoch reference the start of the daily or epoch bucket
  • rewards is an array of objects with four components: numeric , exp and currency. The formula to compute the final rewards in the currency unit is numeric * 10^-exp(text is the string version of this)

Example:

[
    {
        "accounts": [
            "12m7XtVWZQuxTirqtzm2dm5J4r16vSQruPmahThMnskG8Bcp"
        ],
        "chain_id": "mainnet",
        "rewards": [
            {
                "text": "4.43255269 DOT",
                "currency": "DOT",
                "numeric": 44325526900,
                "exp": 10
            }
        ],
        "epoch": "805",
        "network": "polkadot",
        "validator": "138QdRbUTB9eNY94Q4Mj5r39FkgMiyHCAy8UFMNA5gvtrfSB",
        "time": "2022-08-16T15:36:18Z"
    },
    ...
]

Limitations

  • The API returns rewards for Figment validators only
  • The API returns rewards for the last 30 days
  • Data is available approximatively an hour after the end of the day or epoch. When requesting daily for the current day, the data will be available but obviously partially complete as the day progresses.
