Staking API - Polygon

Authentication​

Making API calls with DataHub requires authentication, which can be performed by passing your API key via the request header Authorization or as a parameter in the query URL.

Here's some boilerplate to get you started.

  • Fetch with Header
  • Fetch with Parameter
  • CURL with Header
  • CURL with Parameter
fetch("https://polygon-slate.datahub.figment.io/api/v1/flows", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Authorization" : "<api_key>",
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
fetch("https://polygon-slate.datahub.figment.io/apikey/<api_key>/api/v1/flows", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
curl -X POST 'https://polygon-slate.datahub.figment.io/api/v1/flows' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // JSON Payload
}'
curl -X POST 'https://polygon-slate.datahub.figment.io/apikey/<api_key>/api/v1/flows' \
--header 'Content-Type: application/json' \
--data '{
  // JSON Payload
}'

Available Methods

New Delegation Flow

Create a new flow with a POST request to /api/v1/flows to initiate the staking process.

View specification

Parameters

  • flow - object
    • network_code - string Network this flow operates on (ex. near) (required)
    • chain_code - string Chain this flow operates on (ex. mainnet) (required)
    • operation - string The operation to -perform (ex. staking) (required)
    • version - string API version to use (ex. v1) (optional)

Response

  • id - string ID of the newly created flow.
  • state - string The current state of the flow i.e., initialized.
  • actions - array The actions array contains the name of the next action in the flow and details of the expected inputs in the data field.
Request Body 
{
  "flow": {
    "network_code": "polygon",
    "chain_code": "testnet",
    "operation": "staking",
    "version": "v1"
  }
}

Get Delegation Flow Status

Get the status of existing flow using the flow id from the previous step.

View specification

Parameters

  • flow_id - string Pass the Flow ID you received in the response when creating the flow in the GET request URL, ex. /flows/69d02e2c-bbde-4b09-bbe9-2be287a9a6de.

Response

  • id - string Id of the flow.
  • state - string The current state of the flow, i.e., initialized.
  • actions - array The actions array contains the name of the next action in the flow and details of the expected inputs in the data field.
Request Query 
/api/v1/flows/{{polygonStakingFlowId}}
Example Response

Assign Staking Data

After collecting the required inputs, PUT /api/v1/flows/[:flow_id]/next to move on to the next step in the flow.

View specification

Parameters

* name - string Name of the action to execute (required).
* inputs - array
* delegator_address - string The delegating address (required).
* validator_address - string Address of the validator smart contract you want to delegate your tokens to (required).
* amount - number - The amount of MATIC to be delegated (required).
* minimum_shares_to_mint - string (optional) - the minimum number of shares of the delegation pool to be minted. If the exchange rate results in fewer shares being minted, the transaction will fail.
* max_slippage_percentage - number (optional) - the maximum slippage you are willing to accept.

Response

* id - string Id of the newly created flow.
* state - string Current state of the flow i:e; assign_staking_data
* actions - array The actions array contains the name of the next action in the flow and details of the expected inputs in the data field.

Request Body 
{
  "name": "assign_staking_data",
  "inputs": {
    "delegator_address": null,
    "valiator_address": null,
    "amount": null,
    "minimum_shares_to_mint": null,
    "max_slippage_percentage": null
  }
}
Example Response
""

Refresh Allowance Transaction

Use refresh_allowance_tx if you need to get a fresh version of the transaction Payload (for example, if the nonce needs to be updated).

View specification

Parameters

* name - string Name of the action to execute (required). * inputs - array There are no inputs for this action (optional).

Response

The transaction payload is refreshed.

Request Body 
{
  "name": "refresh_allowance_tx",
  "inputs": null
}
Example Response
""

Sign Allowance Transaction

Submit a signed transaction payload ready to be broadcast to the network.

View specification

Parameters

  • name - string Name of the action to execute (required).
  • inputs - array
    • transaction_payload - string Transaction payload can be found in the response after creating the transaction. (required). OR
    • signatures - array An array of signatures. Staking API will sign the payload if this is provided (optional).

Response

The transaction is broadcast to the network. Once it is confirmed, the staking flow is completed.

Request Body 
{
  "name": "sign_allowance_tx",
  "inputs": {
    "transaction_payload": null,
    "signatures": []
  }
}
Example Response
""
