---
title: Create a New Un-Delegating Flow
---

We will create a new Flow with `POST /api/v1/flows`.

The `flow` JSON field for this request will be:

```json
{
  "network_code": "polygon",
  "chain_code": "testnet",
  "operation": "unstaking",
  "version": "v1"
}
```

```bash title="Example cURL request"
curl -X POST https://polygon-slate.datahub.figment.io/api/v1/flows \
--header 'Authorization: API_KEY' \
--header 'Content-Type: application/json' \
--data '{
  "flow": {
 {
    "network_code": "polygon",
    "chain_code": "testnet",
    "operation": "unstaking",
    "version": "v1"
  }
}'
```

The response indicates one possible `action`:

- `create_unbond_tx` —

The create_unbond_tx action indicates seven fields for data collection:

- `delegator_address` (**required**) — the delegator address
- `validator_address` (**required**) — Validator contract address from which MATIC will be un-delegated
- `amount` (**required**) — the amount of MATIC to be un-delegated
- `max_shares_to_burn` (_optional_) — the maximum number of shares of the delegation to be burned. If the exchange rate results in more shares being burned, the transaction will fail.
- `max_slippage_percentage` (_optional_) — the maximum slippage you are willing to accept
- `gas_limit` (_optional_) — adjust gas limit for the transaction
- `gas_price` (_optional_) — adjust gas price for the transaction
