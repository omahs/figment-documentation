---
title: Submit Deposit Data
---

After collecting the required Inputs, we `PUT /api/v1/flows/[:flow_id]/next` to move on to the next step.

### Parameters

- `name` - `string` Name of the action to execute - `create_transfer_tx`.
- `inputs` - `array`
  - `from_account_pubkey` - `string` The account that will sign the transaction (required).
  - `from_account_address` - `string` the address which NEAR will be transferred from (required).
  - `to_account_address` - `string` The address to which NEAR will be transferred (required).
  - `amount` - `number` The amount of NEAR to be transferred (required).

#### Sample cURL Request

```json
curl -X PUT 'https://near-slate.datahub.figment.io/api/v1/flows/5f0962d5-fba0-453c-bf35-0dad79624b59/next' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // highlight-start
  "name": "create_transfer_tx",
  "inputs": {
    "from_account_address": "slate-demo.testnet",
    "from_account_pubkey": "ed25519:5QA46X6NkNmsFdu9xWVBaLNowh9gGeF1c5r9u6NcxaLY",
    "to_account_address": "pizza.testnet",
    "amount": "10"
  }
  // highlight-end
}'
```

### Response

Note that in this response you can find the unsigned, serialized transaction in `data[create_deposit_transaction][raw]`.

The response indicates two possible `actions` :

- `refresh_deposit_tx` — If you need to get a fresh version of the transaction Payload (i.e., the nonce has increased).
- `sign_deposit_tx` — Submit a signed transaction Payload ready to be broadcast to the network.

We will choose `sign_deposit_tx`, which indicates one Input to collect:

- `transaction_payload` — The signed transaction Payload ready to be broadcast to the network.
