title: Submit Registration Data

---

After collecting the required inputs, send a `PUT` request to `/api/v1/flows/[:flow_id]/next` to proceed to the next step.

### Parameters

- `name` &mdash; `string` Name of the action to execute (required).
- `inputs` &mdash; `object` (required).
  - `delegator_address` &mdash; A Cardano address from which ADA will be staked.
  - `validator_address` &mdash; A Cardano staking pool to which ADA will be staked.

#### Sample cURL Request

```bash {5-11}
curl -X PUT 'https://cardano-slate.datahub.figment.io/api/v1/flows/6f995c8a-08b7-43ff-9275-0ef8fc94bc26/next' \
-H 'Content-Type: application/json' \
-H 'Authorization: <api_key>' \
-d '{
  "name": "create_register_tx",
  "inputs": {
    "delegator_address": "addr_test1qrwsyak374wzmdakesg8wl8r0feucetatxuqz8s2csqzradxfu7p7cvhrg6eh374ps852jxq7qzgfjwgknn36ggkdwrsu6ae9x",
    "validator_address": "pool1wn6a6f23ctq06udwhw27ravdpd6zcr7jlut3yez0wzdackz3222"
  }
}'
```

:::info

A registration is required to complete the flow for the first time. When a registration exists for a delegator address, the flow state will proceed directly from `initialized` to `delegate_tx_signature`.

:::

### Response

The response indicates 3 possible `actions` :

- `refresh_register_tx` &mdash; If you need to get a fresh version of the transaction payload (i.e., the nonce has increased).
- `sign_register_tx` &mdash; Submit a signed transaction payload ready to be broadcast to the network.
- `confirm_register_tx_by_hash` &mdash; Confirm a completed transaction by supplying its hash.

We will choose `sign_register_tx`, which indicates 1 input to collect:

- `transaction_payload` &mdash; The signed transaction payload ready to be broadcast to the network.
