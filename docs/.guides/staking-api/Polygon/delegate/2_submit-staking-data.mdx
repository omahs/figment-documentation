---
title: Submit Staking Data
slug: /guides/staking-api/polygon/delegate/submit-staking-data
---

After creating a new staking flow, submit the staking account data. Collect the required inputs. Send a `PUT` request to `/api/v1/flows/[:flow_id]/next` to proceed to the next step.

### Parameters

- `name` &mdash; `string` Name of the action to execute (required).
- `inputs` &mdash; `object`
  - `delegator_address` &mdash; `string` The delegating address (required).
  - `validator_address` &mdash; `string` The address of the validator smart contract to which you want to delegate your tokens (required).
  - `amount` &mdash; `number` &mdash; The amount of MATIC to be delegated (required).
  - `minimum_shares_to_mint` &mdash; `string` The minimum number of shares of the delegation pool to be minted. If the exchange rate results in fewer shares being minted, the transaction will fail .
  - `max_slippage_percentage` &mdash; `number` The maximum slippage you are willing to accept .

### Sample cURL Request

```bash {5-10}
curl -X PUT 'https://matic-slate.datahub.figment.io/api/v1/flows/536921d0-d0f3-4887-8177-69c855cbcce3/next' \
-H 'Content-Type: application/json' \
-H 'Authorization: <api_key>' \
-d '{
  "name": "assign_staking_data",
  "inputs": {
    "delegator_address": "0x6BeA1177Ff310983f84D4Fc77b7467a3bD5626ed",
    "validator_address": "0x15ED57Ca28cbebb58d9c6C62F570046BC089bC66",
    "amount": "1"
  }
}'
```

### Response

The response indicates two possible `actions`:

- `create_allowance_tx` &mdash; If an allowance transaction is needed to cover the `amount` specified in the staking data, it will be created by the Staking API.
- `assign_staking_data` &mdash; Used to assign new staking data, if needed. Otherwise, previously provided defaults will be used.

**OR**

- `create_delegate_tx` &mdash; If no further allowance is needed for the amount you intend to delegate using the delegator address, you can proceed with the delegate transaction.
- `assign_staking_data` &mdash; Used to assign new staking data, if needed. Otherwise, previously provided defaults will be used.

Assuming that an allowance is needed, we will proceed with `create_allowance_tx` which indicates two optional inputs to collect:

- `gas_limit` &mdash; The gas limit for the transaction.
- `gas_price` &mdash; The highest gas price you will pay for the transaction.
