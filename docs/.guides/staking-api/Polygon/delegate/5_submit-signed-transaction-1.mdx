---
title: Submit a Signed Allowance Transaction for Broadcast
slug: /guides/staking-api/polygon/delegate/submit-signed-allowance-transaction
---

After signing the transaction, send a `PUT` request to `/api/v1/flows/[:flow_id]/next` with the signed payload.
The Staking API will broadcast the transaction to the Polygon network.

### Parameters

- `name` &mdash; `string` Name of the action to execute (required).
- `inputs` &mdash; `object`
  - `transaction_payload` &mdash; Signed transaction payload from the previous step's response.

#### Sample cURL Request

```bash {5-8}
curl -X PUT 'https://matic-slate.datahub.figment.io/api/v1/flows//next' \
-H 'Content-Type: application/json' \
-H 'Authorization: <api_key>' \
-d '{
  "name": "sign_allowance_tx",
	"inputs": {
    "transaction_payload": "0x02f8b005808459682f00850eaa6fba0e82b58394499d11e0b6eac7c0593d8fb292dcbbf815fb29ae80b844095ea7b300000000000000000000000000200ea4ee292e253e6ca07dba5edc07c8aa37a3ffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffc001a0a9cac4be7a0c56d6ed520814958b94d4385b1ac7d791485f85c6c58fa53e0d8aa002f14d836c1b5090b12ef16566056abd57e6fce013a759e444d925b8951633ea"
  }
}'
```

### Response

With the signed transaction provided, the Staking API will broadcast it to the network.
As the transaction progresses towards confirmation, the returned `state` will be either:

- `allowance_tx_broadcasting` &mdash; the transaction has been broadcast but not confirmed.
- `create_delegate_tx` &mdash; the transaction has been confirmed, ready for next step in flow.

Upon finalization of the transaction, the account will be ready for the next step to submit the delegation transaction.
